文件上传

对于文件上传，浏览器在上传的过程中是将 文件以流的形式提交到服务器端的，一般选择采用apache的开源工具common-fileupload这个上传组件。common-fileupload是依赖common-io这个包的，所以还需要下载这个包

文件上传优化

需要用到的类

ServletFileUpload负责处理上传的文件数据，并将表单中的每个输入项封装成一个FileItem对象，在使用ServletFileUpload对象解析请求时需要DiskFileItemFactory对象。所以，我们需要在进行解析工作前构建好DisFileItemFactory对象，通过SerlvetFileUpload对象的构造方法或setFileItemFactory()方法设置ServletUpload对象的fileitemfactory属性。

文件上传要用post请求方式

get方式会有文件大小限制

post方式没有文件大小限制

表单如果包含一个文件上传输入项的话，这个表单的enctype属性就必须设置为multipart/form-data

                    FileOutputStream fos = new FileOutputStream(realPath + "/" + fileName);
                    // 创建一个缓冲区

Cookie(饼干)

会话**：用户打开一个浏览器，并打开了许多的web资源，关闭浏览器，这个过程可以称为会话**

保存用户上次访问的时间**（cookie）**

cookie是一种键值对的形式 name : value.

  • 首先request请求从客户端携带cookie交给服务器端。
  • 在通过循环获取所有的cookie对象，并用cookie的namez作为比较判断，cookie的getValue()方法获得cookie的值
  • 再通过response将服务器自己设置的cookie响应到客户端
  • 通常cookie会有一个有效期可以通过cookie对象调用setMaxAge()方法设置有效期
Cookie[] cookies=request.getCookies();//获取cookie对象
cookie.getName();//获取cookie中的key
cookie.getValue()//获取cookie中vlaue
new Cookie("lastLoginTime",System.currentTimeMillis()+"");//新建一个cookie
cookie.setMaxAge("时间")//cookie的有效期
 response.add(cookie)//响应给客户端一个cookie
 URLenode.encode("中文","utf-8");//给中文编码
  URLDecoder.decode("中文"，"utf-8")//给中文解码

删除cookie

  • 不设置有效期，关闭浏览器cookie会自动消除
  • 将cookie的有效期设置为0
  • cookie.setMaxAge(0)

Cookie细节

Cookie，为什么会被保存下来，一般cookie会保存在本地的用户目录下appdata;

  • cookie是会有上线的大约为300个
  • 一个cookie只能保存一个信息
  • 一个web站点可以给浏览器发送多个cookie,最多存放20个cookie
  • cookie大小有限制为4kb

Session(会话)(重点）

什么是Session

  • 服务器会给每一个用户（浏览器）创建一个Session对象
  • 一个Session独占一个浏览器，只要浏览器没有关闭。这个Session存在；
  • 用户登录后，整个网站他都可以访问！保存用户的信息；保存购物车的信息

Session与Cookie的区别

  • Cookie是把数据写给用户的浏览器，浏览器保存（可以保存多个）

  • Session把用户的数据写到独占Session,服务器端保存（保存重要的信息，减少服务器资源的浪费）

  • Session对象有服务器创建

MVC三层架构(学到后面回来看看就懂了)

MVC: Model View Controller 模型，视图，控制器

Model

  • 业务处理：业务逻辑（Service）
  • 数据持久层CRUD（Dao）

View

  • 展示数据
  • 提供链接发起的Servlet请求（a，form，img...）

Controller（Servlet）

  • 接收用户的请求：（req:请求参数，Session信息...）
  • 交给业务层处理对应的代码
  • 控制视图的跳转 
    登录--->接收用户的登录请求--->处理用户的请求（获取用户登录的参数，username，password）---->交给业务层处理登录业务（判断用户名密码是否正确：事务）--->Dao层查询用户名和密码是否正确-->数据库
